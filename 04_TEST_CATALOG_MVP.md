# Mindletics — каталог тестов для MVP

## Общий принцип

Сайт должен быть современным и стильным. Нам нравится https://hyrox.com/elite-15-qualification/

Все тесты должны быть:
- без ввода текста;
- без печати слов;
- без микрофона;
- без свободного ответа;
- только нажатия пальцем;
- выбор из 3–5 вариантов или нажатие по объектам на экране.
- БЕЗ картинок (у нас нет файлов). Картинки можешь нарисовать сам.

---

# Блок 1. Логика / абстрактное мышление

## 1.1. Числовые ряды
### Суть
Показать последовательность чисел и 4 варианта ответа. Нужно выбрать следующее число.

### Примеры
- 3, 6, 9, 12, ? → 15
- 2, 4, 8, 16, ? → 32
- 5, 7, 10, 14, 19, ? → 25

### UI
- сверху последовательность;
- снизу 4 большие кнопки с вариантами.

### Что логировать
- sequence
- options
- correct_option
- selected_option
- response_time_ms

---

## 1.2. Фигурные последовательности
### Суть
Показать ряд фигур, меняющихся по правилу, и 4 варианта следующей фигуры.

### Примеры правил
- поворот;
- зеркальность;
- добавление элементов;
- изменение цвета / размера / позиции.

### UI
- сверху ряд фигур;
- снизу 4 карточки-варианта.

### Что логировать
- prompt_figure_set
- options
- correct_option
- selected_option
- response_time_ms

---

## 1.3. Простые матрицы
### Суть
Показать простую визуальную матрицу 2x2 или 3x3 с пропуском и 4 варианта ответа.

### UI
- матрица по центру;
- недостающий элемент отмечен пустой ячейкой;
- внизу 4 варианта.

### Что логировать
- matrix_payload
- options
- correct_option
- selected_option
- response_time_ms

---

# Блок 2. Память

## 2.1. Запоминание объектов
### Суть
На 3–4 секунды показать 5–6 объектов. Затем показать расширенный набор и попросить выбрать те, которые были.

### UI
- экран просмотра;
- затем экран выбора объектов.

### Что логировать
- shown_items
- selectable_items
- selected_items
- correct_count
- wrong_count
- response_time_ms

---

## 2.2. Запоминание позиций в сетке
### Суть
На сетке 4x4 на короткое время подсвечиваются несколько клеток. Потом участник должен нажать те же клетки.

### UI
- крупная сетка;
- после исчезновения подсветки пользователь нажимает клетки.

### Что логировать
- shown_cells
- selected_cells
- exact_match
- response_time_ms

---

## 2.3. Последовательность объектов
### Суть
Несколько фигур появляются по очереди. Затем участник должен нажать их в том же порядке.

### UI
- зона анимации;
- затем панель с 4–5 фигурами;
- участник нажимает последовательность.

### Что логировать
- shown_sequence
- selected_sequence
- exact_match
- response_time_ms

---

# Блок 3. Скорость реакции / когнитивный контроль

## 3.1. Choice Reaction
### Суть
Появляется стимул, например цветной квадрат. Нужно нажать соответствующую кнопку.

### UI
- крупный стимул по центру;
- снизу 3–4 большие кнопки.

### Что логировать
- stimulus
- options
- selected_option
- correct_option
- response_time_ms

---

## 3.2. Go / No-Go
### Суть
Если появляется целевой стимул — нужно нажать кнопку. Если нет — не нажимать.

### UI
- по центру стимул;
- одна большая кнопка действия;
- автоматическая фиксация отсутствия нажатия.

### Что логировать
- stimulus
- is_target
- user_action
- was_correct
- response_time_ms

---

## 3.3. Stroop
### Суть
Показывается слово цвета, написанное другим цветом. Нужно выбрать цвет чернил.

### UI
- по центру слово;
- снизу 3–5 цветных кнопок.

### Что логировать
- word_text
- ink_color
- selected_color
- was_correct
- response_time_ms

---

# Блок 4. Финальный визуальный блок

## 4.1. Поиск символа
### Суть
Показать целевой символ и несколько похожих вариантов. Нужно выбрать правильный.

### UI
- сверху target;
- снизу 4 варианта.

### Что логировать
- target
- options
- selected_option
- correct_option
- response_time_ms

---

## 4.2. Сравнение пар
### Суть
Показать две строки или две картинки. Нужно ответить: одинаковые они или нет.

### UI
- две области сравнения;
- две большие кнопки: «Да» / «Нет».

### Что логировать
- left_payload
- right_payload
- correct_answer
- selected_answer
- response_time_ms

---

## 4.3. Визуальная аналогия
### Суть
Показать визуальную пару A:B и вторую пару C:?. Нужно выбрать недостающий элемент из 4 вариантов.

### UI
- сверху схема аналогии;
- снизу 4 варианта.

### Что логировать
- analogy_payload
- options
- correct_option
- selected_option
- response_time_ms

---

# Рекомендуемая комплектация блоков

## Блок 1. Логика
- 2 числовых ряда
- 2 фигурных последовательности
- 2 простые матрицы

## Блок 2. Память
- 2 задания на объекты
- 2 задания на сетку
- 2 задания на последовательность

## Блок 3. Реакция
- 2 choice reaction
- 2 go/no-go
- 2 Stroop

## Блок 4. Финальный визуальный
- 2 поиска символа
- 2 сравнения пар
- 2 визуальные аналогии

Итого: **24 задания на весь сценарий**.

---

# Рекомендация по MVP

Если нужно ещё упростить первую версию:
- делать не по 6, а по **5 заданий в каждом блоке**;
- сначала реализовать по **2 типам тестов на блок**, а третий добавить позже.
